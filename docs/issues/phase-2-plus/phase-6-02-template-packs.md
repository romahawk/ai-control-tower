# [Phase 6] Add reusable template packs for scenarios, workflows, prompts, and tools

GitHub Issue: `#13`  
Status: `Open`

## Goal
Package seeded systems into reusable template packs.

## Why
Templates turn the current hardcoded system into a repeatable product asset and shorten onboarding.

## Scope
- Define template-pack structure
- Bundle scenarios, workflows, prompts, and tools
- Support duplication into user-owned records later

## Out of Scope
- Public marketplace
- Paid template distribution
- Team sharing
- Backend publishing

## Acceptance Criteria
- [ ] Template pack structure exists for seeded systems
- [ ] Seeded records can be represented inside packs
- [ ] Pack duplication path is defined for later implementation
- [ ] Build/lint passes where relevant

## User / Output Value
Future users can start from proven systems instead of building everything from zero.
