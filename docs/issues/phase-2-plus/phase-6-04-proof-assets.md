# [Phase 6] Add proof and case-study integration for reusable outputs

GitHub Issue: `#15`  
Status: `Open`

## Goal
Turn outputs into reusable proof assets such as case studies, portfolio entries, or execution evidence.

## Why
Outputs are more valuable when they can be reused as proof for clients, jobs, or product credibility.

## Scope
- Define proof asset records derived from outputs
- Connect outputs to proof-asset selection flows
- Reuse case-study workflows as the first proof path

## Out of Scope
- CMS backend
- Public publishing pipeline
- SEO work
- Marketing site rebuild

## Acceptance Criteria
- [ ] Outputs can be flagged for proof reuse
- [ ] A proof-asset record can be created from workflow outputs
- [ ] Case-study workflows connect to the proof flow
- [ ] Build/lint passes where relevant

## User / Output Value
User can turn execution into visible credibility instead of leaving outputs trapped inside the workflow system.
